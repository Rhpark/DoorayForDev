- Dooray 관련 요청을 받았을 때만 루트의 `DOORAY.md`를 읽고, 해당 문서와
  `.agents/skills/dooray-*`의 지침을 따른다. 일반 작업에서는 `DOORAY.md`를 읽지 않는다.

- Dooray 번호가 있는 규모 있는 구현 요청은 바로 구현하지 않는다.
  먼저 `$dooray-report`로 적용 방안을 작성하도록 안내하고,
  구현은 사용자의 별도 명시적 요청을 받은 뒤 진행한다. 자잘한 수정은 예외로 한다.
  상세 절차는 `.agents/skills/dooray-*`를 진실의 원천으로 삼는다.

- `docs/agents/plan/` 아래 문서는 업무 기록이므로 덮어쓰거나 삭제하지 않는다.
  수정이 필요하면 날짜 섹션 추가 등 기존 이력을 보존하는 방식으로 내용을 추가한다.

- 질문에는 요청한 범위만 답하고 작업한다.
  요청 범위 밖의 조사나 수정이 필요하면 먼저 사용자에게 알리고 지시를 기다린다.
