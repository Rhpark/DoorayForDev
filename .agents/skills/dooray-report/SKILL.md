# Dooray 적용 방안 보고서

Dooray 이력을 이해하고 현재 저장소 코드에 어떻게 적용하거나 해결할지 문서화한다. 구현은 하지 않는다.

사용자가 이 스킬을 직접 호출한 것은 다음 파일 작업만 승인한 것으로 본다.

- 분석에 필요한 첨부파일을 `Dooray/report/<업무번호>/download/`에 저장
- `Dooray/report/<업무번호>/REPORT.md`를 생성하거나 갱신

그 외 소스 코드·설정 수정, Git 커밋, Dooray 댓글 등록·상태 변경은 금지한다.

문서 전체에 이력 본문·댓글·첨부파일·실제 저장소 코드에서 확인한 사실만 쓴다. 확인되지 않은 내용은 추측하지 말고 `열린 질문 / 확인 필요` 또는 `리스크 / 미확인 사항`으로 분류한다.

## 1. 이력과 첨부 확인

저장소 루트에서 이력 전체를 조회한다.

```
python Dooray/dooray.py full <업무번호>
```

`full` 원본은 분석 자료로만 사용하고 사용자 응답에 그대로 복사하지 않는다. 태그가 있으면 작업 성격을 판단하는 보조 근거로 사용하되 본문과 댓글의 구체적인 요구사항을 우선한다.

첨부파일이 요구사항에 직접 언급됐거나 구현 판단에 필요하면 다음 명령으로 다운로드한다.

```
python Dooray/dooray.py download <업무번호> <파일명>
```

- 다운로드 경로는 `Dooray/report/<업무번호>/download/<파일명>`이다.
- `.md`, `.csv`, `.json`은 텍스트로 읽고 `.png`, `.jpg`, `.jpeg`는 이미지 보기 도구로 확인한다.
- `.pdf`는 현재 환경에서 직접 내용을 확인할 수 있을 때만 분석한다.
- 직접 확인할 수 없는 `.pptx`, `.docx`, `.xlsx` 등의 형식은 다운로드하거나 추측하지 말고 `리스크 / 미확인 사항`에 기록한다.
- 확인한 첨부파일명과 그 파일에서 확인한 사실을 `분석`에 근거로 남긴다.

## 2. 기존 보고서와 코드 탐색

`Dooray/report/<업무번호>/REPORT.md`가 있으면 먼저 읽는다.

이력과 댓글에서 뽑은 키워드로 `rg`를 사용해 관련 정의, 호출부, 설정과 테스트를 찾는다. 필요한 경우 명백한 동의어와 진입점도 확인한다. 각 요구사항마다 다음 중 하나를 기록하면 탐색을 중단한다.

- 실제 관련 코드를 찾고 `파일경로`의 `심볼명(인자)` 근거를 기록
- 관련 코드를 찾지 못하고 검색한 키워드와 범위를 함께 기록

요구사항과 무관한 영역까지 통독하지 않는다. 여러 독립 영역에 걸쳐 상세 매핑이 과도하면 영향 영역만 상위 수준으로 정리하고 상세 설계와 구현은 별도 사용자 요청이 필요하다고 명시한다.

방안에 등장하는 기존 코드는 실제로 확인한 것만 근거로 든다. 함수가 아닌 설정 키·상수·CLI 하위 명령은 이름을 그대로 쓴다. 새 파일이나 새 심볼이 필요한 방안은 `신규 생성 제안`으로 명확히 구분하며 존재하는 것처럼 쓰지 않는다.

## 3. 보고서 작성 또는 갱신

보고서에 다음 섹션을 한국어로 작성한다.

- **문서 정보**: 이력번호 / 최초 작성 시각 / 최종 갱신 시각
- **이력 요약**: 번호 / 제목 / 현재 상태 / 태그 / 핵심 요구사항
- **분석**: 무엇을 해야 하는지와 그 이유, 이력·댓글·첨부 근거
- **이 프로젝트에서의 적용 방안**: 대상 영역·파일과 접근 순서, 확인한 코드 근거
- **열린 질문 / 확인 필요**: 사람이 결정해야 하는 질문과 각 질문 아래의 `→ 답변:` 빈칸. 없으면 `없음`
- **리스크 / 미확인 사항**: 확인하지 못한 첨부나 코드 근거, 위험 요소
- **갱신 이력**: 갱신 시각과 이전 결론에서 변경·제거한 내용 및 이유

기존 보고서가 있으면 새 보고서를 뒤에 통째로 붙이지 않는다. 본문은 최신 분석으로 갱신하되 최초 작성 시각과 기존 갱신 이력은 유지한다. 기존 결론을 변경하거나 제거하거나 열린 질문에 답이 추가되면 그 이유를 새 갱신 이력으로 남긴다.

PowerShell에서 `Get-Date -Format "yyyy-MM-dd HH:mm"`으로 현재 시각을 구한다. 처음 작성할 때는 최초 작성과 최종 갱신 시각을 현재 시각으로 기록한다. 갱신할 때는 최초 작성 시각을 유지하고 최종 갱신 시각만 바꾼다.

`apply_patch`를 사용해 다음 파일을 생성하거나 갱신한다.

```
Dooray/report/<업무번호>/REPORT.md
```

## 4. 결과 전달

저장한 파일 경로를 알려주고 저장된 `REPORT.md` 내용을 요약하지 말고 그대로 보여준다. 실제 구현은 별도의 명시적인 사용자 요청이 있을 때만 진행한다.
